# 学习笔记整理助手

## 功能概述

自动分析学习材料，提取关键信息，并生成结构化的Markdown格式笔记。

## 核心能力

### 1. 知识点提取
- 识别核心概念和术语
- 提取重要原理和公式
- 总结关键结论

### 2. 重点标注
- 标记高频考点
- 识别易错点和注意事项
- 突出实践应用场景

### 3. 定义整理
- 提取专业术语定义
- 整理概念对比表格
- 建立术语索引

### 4. 结构化输出
- 按章节/主题分类
- 生成目录导航
- 统一格式规范

## 使用方法

当用户需要整理学习笔记时：

1. **接收输入**：获取原始学习材料（文本、截图、文档等）
2. **分析内容**：
   - 识别材料类型（教材/讲义/视频笔记/阅读笔记）
   - 确定学科领域
   - 理解学习目标
3. **提取信息**：
   - 扫描核心知识点
   - 标记重点和难点
   - 整理定义和概念
4. **生成输出**：
   - 创建结构化Markdown
   - 按章节/主题组织
   - 添加适当的格式和标记

## 输出格式模板

```markdown
# [主题/章节标题]

## 目录
- [核心概念](#核心概念)
- [重点知识](#重点知识)
- [定义术语](#定义术语)
- [详细内容](#详细内容)

## 核心概念
- 概念1：简要说明
- 概念2：简要说明

## 重点知识
### 🔴 高频考点
1. 考点1
2. 考点2

### ⚠️ 易错点
- 易错点1
- 易错点2

## 定义术语
| 术语 | 定义 | 备注 |
|------|------|------|
| 术语1 | 定义内容 | 补充说明 |

## 详细内容
### [子主题1]
...

### [子主题2]
...

---
*整理时间：YYYY-MM-DD*
*来源：[原始材料来源]*
```

## 标记规范

- 🔴 高频考点 / 核心重点
- 🟡 重要概念 / 需要理解
- 🟢 一般了解 / 扩展知识
- ⚠️ 易错点 / 注意事项
- 💡 实践技巧 / 应用场景
- 🔗 相关链接 / 延伸阅读

## 示例

**输入**：
"JavaScript的闭包是指有权访问另一个函数作用域中的变量的函数。创建闭包的常见方式，就是在一个函数内部创建另一个函数。闭包会导致内存泄漏问题，因为被引用的变量不会被垃圾回收。"

**输出**：
```markdown
# JavaScript 闭包

## 核心概念
- **闭包(Closure)**：有权访问另一个函数作用域中变量的函数

## 重点知识
### 🔴 高频考点
1. 闭包的定义和特性
2. 闭包的创建方式
3. 闭包的内存影响

### ⚠️ 易错点
- 闭包会导致内存泄漏，被引用的变量不会被垃圾回收

## 定义术语
| 术语 | 定义 | 备注 |
|------|------|------|
| 闭包 | 有权访问另一个函数作用域中变量的函数 | 函数内部创建函数 |

## 详细内容
### 创建方式
在函数内部创建另一个函数

### 注意事项
- 内存管理：闭包引用的变量不会被垃圾回收
- 可能导致内存泄漏
```

## 最佳实践

1. **保持简洁**：每个知识点控制在3-5句话
2. **层次分明**：使用标题层级清晰组织
3. **表格辅助**：对比类内容使用表格呈现
4. **代码高亮**：技术类笔记使用代码块
5. **链接关联**：相关知识点建立交叉引用
